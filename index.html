User Recommendation System

A user recommendation system that can return top 5 statements based on user query and filtering from Redis database leveraging OpenAI, Cohere, and Sbert models. Automated data pipeline with airflow DAGs and deployed a dockerized app with FastAPI backend, Redis database, and Streamlit user interface

    Details

    Case study

    Intelligence Co is tracking advances in Large language models and is interested in trying out a pilot project that involves building a contextual search application that leverages vector similarity search, traditional filtering and hybrid search features to aid financial researchers search through earnings call transcripts. They envision an application wherein they can periodically upload text datasets and have investment analysts being able to filter by company and year and have investment analysts being able to search through earnings call transcripts to aid research.

    Tech stack

    • Redis - Stored hash keys of companies earnings statement and its embeddings using OpenAI and SBert.
    • FastAPI - User registration, authentication, and login.
    • Streamlit - User interface where users can perform hybrid filtering and querying to get the result.
    • Airflow - Data pipeline automation using DAGs to retreive data source from github and load it into redis.
    • Docker - To create a dockerized app of Redis, Streamlit, FastAPI, and Airflow.
    • Postman - To check the health of FastAPI endpoints and to send POST/GET requests.
    • OpenAI text-embedding-ada-002 model - is used to compute embeddings for each chunk of the earnings transcript and add them as a column in Redis.
    • SBERT sentence-transformers/all-MiniLM-L6-v2 model - is used to compute embeddings for each chunk of the earnings transcript and add them as a column in the database.
